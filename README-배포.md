# Vercel 배포 메모

## 현재 구조
- `index.html`
- `styles.css`

지금 구조는 정적 사이트라서 Vercel에 바로 올릴 수 있습니다.

## 왜 이 방식이 맞는가
- 현재 범위가 메인 시안 중심이라 Next.js까지 바로 올릴 필요가 없음
- 다른 사람에게 링크를 보여주고 수정 피드백 받기 쉬움
- 나중에 필요해지면 Next.js로 옮겨도 부담이 적음

## 추천 배포 방식
1. 이 폴더만 별도 Git 저장소로 관리
2. GitHub에 업로드
3. Vercel에서 저장소 연결
4. 배포 후 생성된 링크 공유

## 만약 현재 Obsidian 전체 저장소를 연결한다면
- Vercel 프로젝트의 Root Directory를 `AI/홈페이지`로 지정

## 나중에 Next.js로 옮겨야 하는 시점
- 문의 폼 백엔드 처리
- 다국어 페이지 분기
- 관리자 업데이트
- PayPal 같은 결제 흐름
